# リポジトリ別記事一覧機能 要件定義書

**作成日**: 2026年1月10日
**更新日**: 2026年1月10日

## 1. 機能概要

公開画面において、特定のリポジトリに紐づく記事のみを一覧表示する。

**背景**：
- 現状、公開画面のトップページでは全リポジトリの記事が混在して表示されている
- 特定のリポジトリに関する記事だけを確認する手段がない

**主要な提供価値**：
- リポジトリ単位での記事閲覧が可能になる
- 興味のあるリポジトリの更新情報を効率的に追跡できる

## 2. 機能要件

### 2.1 リポジトリ別記事一覧ページ

- 指定されたリポジトリに紐づく公開済み記事を一覧表示する
- 既存のトップページ（全記事一覧）と同様の表示形式とする

### 2.2 導線

#### トップページからの導線

- トップページにリポジトリ一覧を表示する
- 各リポジトリ名をクリックすると、そのリポジトリの記事一覧ページへ遷移する

#### 記事詳細ページからの導線

- 記事詳細ページにリポジトリ名を表示する
- リポジトリ名をクリックすると、そのリポジトリの記事一覧ページへ遷移する

## 3. 前提条件・制約事項

### 3.1 前提条件

- Repositoryモデルが存在し、Postモデルと1対多の関連を持つこと
- Postモデルにstatus属性（draft/published/skipped）が存在すること

### 3.2 制約事項

- 公開済み（published）の記事のみを表示対象とする
- 管理画面は本機能の対象外とする

## 4. 用語定義

| 用語 | 説明 |
|------|------|
| リポジトリ | GitHub上のリポジトリ。記事生成の対象となるPRの取得元 |
| 記事 | PRの変更内容を要約した公開コンテンツ（Postモデル） |
| 公開済み | statusがpublishedの記事 |

## 5. 受け入れ条件

- [ ] リポジトリ別の記事一覧ページが存在すること
- [ ] 指定したリポジトリに紐づく公開済み記事のみが表示されること
- [ ] トップページからリポジトリを選択して記事一覧ページへ遷移できること
- [ ] 記事詳細ページからリポジトリ名をクリックして記事一覧ページへ遷移できること
